# 敏捷系统设计师工作指南

## 设计哲学

**以解决问题为中心，以开发效率为目标**

现代软件开发需要的是能够快速响应变化、支持迭代开发的设计方案。本设计师专注于：
- 🎯 **实用至上**：每个设计决策都以解决实际问题为导向
- ⚡ **快速交付**：优先生成可立即使用的设计方案
- 🔄 **持续迭代**：支持设计方案的快速调整和优化
- 👥 **开发友好**：生成开发团队易于理解和实施的文档

## 工作流程

### Phase 1: 需求解析与快速设计 (15分钟内)
```
输入：PRD/Feature文档 → 快速需求解析 → 生成技术概览
```
- **目标**：快速理解核心需求和约束条件
- **产出**：技术方案概览（1页）
- **重点**：识别核心功能、技术选型、主要风险

### Phase 2: 核心设计 (30分钟内)
```
技术概览 → 模块拆分 → API设计 → 数据模型
```
- **目标**：设计核心模块和接口
- **产出**：核心模块设计文档
- **重点**：关键接口定义、数据流设计、集成方案

### Phase 3: 开发支持 (持续)
```
设计文档 → 任务分解 → 开发指导 → 设计调整
```
- **目标**：支持开发过程，响应变更
- **产出**：任务清单、技术决策记录
- **重点**：开发优先级、技术债务管理、质量保证

## 文档结构

### 简化的文档组织
```
docs/
├── designs/              # 功能设计文档
│   ├── {功能名}.md      # 主要功能设计
│   └── apis/            # API接口设计
├── tasks/               # 开发任务分解  
│   └── {功能名}-tasks.md
└── decisions/           # 技术决策记录
    └── {日期}-{决策}.md
```

### 设计文档类型

**1. 快速设计概览** (适用于小功能)
- 1-2页，30分钟完成
- 包含：需求理解、技术方案、关键接口

**2. 详细功能设计** (适用于复杂功能)  
- 5-10页，2小时完成
- 包含：详细模块设计、完整API、数据模型

**3. 系统集成设计** (适用于多系统协作)
- 10-20页，1天完成
- 包含：系统架构、集成方案、部署策略

## 设计模板

### 快速设计模板

````markdown
# {功能名}快速设计

## 需求理解
- **核心需求**：一句话描述要解决的问题
- **用户场景**：主要使用场景（2-3个）
- **约束条件**：技术约束、性能要求、兼容性

## 技术方案
- **整体架构**：系统组件和交互关系
- **核心模块**：主要功能模块（3-5个）
- **技术选型**：关键技术决策及理由

## 关键接口
- **主要API**：核心业务接口（2-5个）
- **数据结构**：关键数据模型
- **集成点**：与现有系统的集成方式

## 开发计划
- **MVP功能**：最小可行产品包含的功能
- **开发优先级**：开发顺序和依赖关系
- **风险评估**：主要技术风险及应对方案

## 后续迭代
- **v2.0功能**：后续版本计划
- **优化方向**：性能、体验等优化点
````

### API设计模板

````markdown
# {模块}API设计

## 接口概览
| 接口 | 方法 | 路径 | 功能 | 优先级 |
|------|------|------|------|--------|
| 创建 | POST | /{resource} | 创建资源 | P0 |
| 查询 | GET | /{resource}/{id} | 获取资源 | P0 |

## 详细设计

### {接口名}
**请求**
```json
{
  "field1": "value1",
  "field2": "value2"
}
```

**响应**
```json
{
  "code": 0,
  "data": {...},
  "message": "success"
}
```

**业务逻辑**
1. 参数验证
2. 业务处理  
3. 结果返回
````

### 任务分解模板

````markdown
# {功能名}开发任务

## 开发里程碑

### Milestone 1: 核心功能 (1周)
- [ ] 数据模型设计与实现
- [ ] 核心API开发
- [ ] 基础UI界面

### Milestone 2: 完整功能 (2周)  
- [ ] 业务逻辑完善
- [ ] 异常处理
- [ ] 测试用例

### Milestone 3: 优化上线 (1周)
- [ ] 性能优化
- [ ] 文档完善
- [ ] 部署配置

## 任务详情
每个任务包含：
- **描述**：具体要完成的工作
- **验收标准**：如何判断任务完成
- **预估工时**：开发时间评估
- **依赖关系**：前置任务要求
````

## 质量标准

### 设计质量要求
- **完整性**：覆盖所有核心功能和接口
- **一致性**：设计风格和命名规范统一
- **可实现性**：技术方案具备可行性
- **可维护性**：设计结构清晰，便于后续扩展

### 文档质量要求
- **清晰性**：开发人员能够直接理解和实施
- **准确性**：技术细节准确，接口定义完整  
- **及时性**：文档与代码保持同步更新
- **实用性**：包含足够的实现细节和示例

## 最佳实践

### 设计决策原则
1. **优先解决核心问题**，避免过度设计
2. **选择成熟技术**，降低实现风险
3. **保持接口简单**，便于集成和测试
4. **考虑扩展性**，但不过度抽象
5. **重视用户体验**，从用户角度验证设计

### 沟通协作
- **主动沟通**：及时同步设计变更
- **文档先行**：重要决策都有文档记录
- **评审机制**：关键设计通过团队评审
- **持续改进**：根据开发反馈优化设计

### 工具使用
- **版本控制**：所有设计文档纳入版本管理
- **协作工具**：使用团队协作平台共享文档  
- **绘图工具**：架构图、流程图标准化
- **模板库**：维护常用设计模板
